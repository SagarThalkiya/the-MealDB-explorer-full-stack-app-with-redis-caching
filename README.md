# TheMealDB Explorer

A full-stack playground that wraps [TheMealDB](https://www.themealdb.com/api.php) behind a FastAPI proxy with caching and exposes a modern React UI for searching, browsing, and inspecting meals.

## One-Time Setup

```powershell
# From the repo root
python -m venv .venv
.\\.venv\\Scripts\\activate
pip install -r backend\\requirements.txt

cd frontend
npm install
```

## Running the stack

### Backend (FastAPI)

```powershell
.\\.venv\\Scripts\\activate
uvicorn app.main:app --reload --app-dir backend --port 8000
```

### Frontend (React/Vite)

```powershell
cd frontend
npm run dev      # http://localhost:5173
npm run build    # production bundle in frontend/dist
npm run preview  # serve the production bundle locally
```

The Vite dev server proxies `/api` calls to `http://localhost:8000` (see `frontend/vite.config.ts`), so both processes must run simultaneously during development.

## Configuration

| Variable             | Where           | Default | Description                                    |
|----------------------|-----------------|---------|------------------------------------------------|
| `MEALDB_API_KEY`     | Backend env var | `1`     | TheMealDB key (public test key by default)     |
| `CACHE_TTL_SECONDS`  | Backend env var | `300`   | In-memory cache expiry for upstream responses  |
| `CACHE_MAX_SIZE`     | Backend env var | `256`   | Maximum cached entries                         |
| `VITE_API_BASE_URL`  | Frontend env    | `/api`  | Override API base URL when deploying           |

## API Reference (all under `/api`)

| Method | Endpoint                          | Purpose                                  |
|--------|-----------------------------------|------------------------------------------|
| GET    | `/api/health`                     | Service heartbeat                        |
| GET    | `/api/meals/search?q=term`        | Search meals by partial name             |
| GET    | `/api/meals/{meal_id}`            | Fetch a single meal with ingredients     |
| GET    | `/api/meals/random`               | Fetch a random meal                      |
| GET    | `/api/categories`                 | List meal categories                     |
| GET    | `/api/categories/{category}/meals`| Meals that belong to the category        |

Responses are normalized into compact Pydantic schemas tailored for the UI (consistent field names, flattened ingredients, cleaned metadata).

## Frontend Experience

- **Global search** with optimistic loading and friendly empty states
- **Category browser** using a dropdown + responsive grid
- **Random meal** CTA that surfaces a new recipe instantly
- **Recipe detail panel** with ingredients, instructions, and outbound links
- Fully responsive layout with modern styling (Inter font stack, cards, banners)

## Caching & Resilience

- `cachetools.TTLCache` guards every outbound call to TheMealDB (size + TTL configurable)
- Graceful fallbacks for empty or malformed upstream payloads
- Centralized FastAPI router with consistent HTTP errors (404, 502 where relevant)

## Deployment Tips

- Serve the backend (uvicorn/gunicorn) and the static frontend separately, or let any reverse proxy route `/api` to FastAPI and everything else to the built `frontend/dist`.
- If you deploy the frontend under a different origin, set `VITE_API_BASE_URL` at build time (e.g., `VITE_API_BASE_URL=https://yourdomain/api npm run build`).
- Consider replacing the in-memory cache with Redis for multi-instance deployments—the interface is isolated in `backend/app/cache.py`.
